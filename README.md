# Moscow Flat Prices by Districts

Интерактивное приложение на **Streamlit** для анализа и визуализации цен на квартиры в Москве по районам.

## Что делает приложение

Приложение показывает динамику цены за квадратный метр по районам и помогает сравнивать районы между собой:

- строит карту районов Москвы (choropleth) с выбранной метрикой;
- показывает текущую цену за м²;
- считает изменение цены на окнах `4, 8, ..., 32` квартала:
  - абсолютное изменение (`abs_change_*q`),
  - относительное изменение в % (`pct_change_*q`),
  - CAGR в % (`cagr_*q`);
- учитывает фильтр по минимальному числу сделок (`n_deals`) для надежности данных;
- автоматически пытается сопоставить названия районов между панелью и GeoJSON.

## Какие данные нужны

### 1) Входной CSV для препроцессинга

Для скрипта `preprocessing.py` нужен CSV минимум с колонками:

- `district_name` — название района;
- `quarter` — текст квартала (например, `2024Q1`);
- `year` — год (число);
- `q_num` — номер квартала `1..4`;
- `n_deals` — число сделок;
- `price_per_m2` — цена за м².

После препроцессинга в Parquet добавляются лаговые и производные метрики, которые использует приложение.

### 2) Границы районов (GeoJSON)

Файл `moscow_districts.geojson` должен содержать полигоны районов. Желательно иметь колонку `district_name`, но приложение также умеет подобрать подходящую строковую колонку автоматически.

### 3) Панель для приложения (Parquet)

Файл `panel.parquet` — результат работы `preprocessing.py`. Именно его читает `app.py`.

## Как подключать данные

### Шаг 1. Подготовьте CSV

Можете взять шаблон из репозитория:

- **пример файла для загрузки:** `examples/upload_example.csv`
- **полный путь в проекте:** `/workspace/moscow_flat_prices_by_districts/examples/upload_example.csv`

### Шаг 2. Препроцессинг

```bash
python preprocessing.py examples/upload_example.csv panel.parquet --min-deals 5
```

Если у вас свой файл, подставьте его путь вместо `examples/upload_example.csv`.

### Шаг 3. Подключите GeoJSON

Положите файл границ районов в корень проекта с именем:

- `moscow_districts.geojson`

### Шаг 4. Запуск приложения

```bash
streamlit run app.py
```

После запуска приложение прочитает:

- `panel.parquet`
- `moscow_districts.geojson`

из корня проекта.

## Структура проекта

- `app.py` — Streamlit-приложение (карта, метрики, фильтры);
- `preprocessing.py` — подготовка панели из CSV в Parquet;
- `examples/upload_example.csv` — пример входного файла для загрузки.

---

## Примечания по качеству данных

- В `preprocessing.py` проверяется уникальность ключа `district_name × quarter`;
- Некорректные `year`/`q_num` приведут к ошибке;
- Метрики роста считаются только при валидной и положительной базовой цене;
- Для надежности можно настраивать `--min-deals`.
